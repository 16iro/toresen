# Toresen MVP 및 마일스톤

이 문서는 **무엇을 먼저 만들지와 그 순서**를 정의한다. 시스템 동작 방식은 [DESIGN.md](DESIGN.md), 제품 정의는 [SPEC.md](SPEC.md)를 참고한다.

---

## 1. MVP 목표

한 개의 그래픽스 트랙을 끝까지 돌릴 수 있는 **자기 훈련 루프**를 자기 컴퓨터(로컬호스트)에서 동작시킨다. 즉, AI가 과제를 발급하고 → 사용자가 코드를 push하면 → 코드 리뷰와 렌더 검증을 받고 → 통과하면 다음 과제로 넘어가는 사이클이 실제로 한 바퀴 이상 돈다.

성공의 기준은 화려함이 아니라 **"이 도구만으로 실제 그래픽스 과제(예: 레이캐스터)를 끝까지 해냈다"**이다.

---

## 2. MVP 범위

**포함**
- 단일 그래픽스 트랙 1개 (예: 레이캐스터 단계)
- 단일 머신, 로컬호스트 실행
- 훈련 루프: 과제(Issue) → push → PR → 리뷰 → 코멘트 → 합격 게이트 → 수동 머지 → 다음 과제
- 코드 리뷰(본인 CLI) + 기계적 1차 검사(빌드·금지 기능)
- 렌더 검증(환경 고정 + 캡처 + SSIM)과 정답 자료 생성
- 봇 계정 + GitHub 연동(Issue·PR·코멘트)
- 상태 저장(`.toresen/state.db`)과 기본 대시보드(리뷰 미러 + 진척)

**제외 (이후로)**
- 다중 트랙·다중 언어·다중 사용자
- 두 기기로 물리적 분리
- 본격 게이미피케이션
- 봇 계정·저장소 자동 생성 (MVP는 사용자가 한 번 수동으로)
- 잠그기 어려운 과제의 멀티모달 시각 평가

---

## 3. 빌드 마일스톤

이른 단계일수록 수동·하드코딩 지름길을 허용하고, 뒤로 갈수록 자동화한다. 각 단계는 그 자체로 돌려볼 수 있는 단위다.

### 마일스톤 0 — 사전 스파이크

가장 불확실한 부분을 먼저 확인한다.

- 화면 없는 컨테이너에서 GPU로 한 프레임을 렌더해 이미지 파일로 저장되는지 확인한다.
- 본인 LLM CLI를 호출해 정해진 형식(JSON)의 답을 받아오는지 확인한다.

**완료 기준:** 두 가지가 각각 동작하는 최소 예제가 돈다.

### 마일스톤 1 — 루프 골격

GitHub·봇·상태의 뼈대를 세운다. 리뷰 품질은 아직 신경 쓰지 않는다.

- 수동으로 만든 학습용 저장소 + 봇 협업자, 하드코딩한 과제 1개로 시작한다.
- 과제를 Issue로 발급 → PR을 폴링으로 감지 → 더미 코멘트 작성 → 상태 DB 갱신.

**완료 기준:** 더미 과제가 발급되고, PR을 올리면 봇이 코멘트를 달고 상태가 바뀐다.

### 마일스톤 2 — 코드 리뷰

실제 리뷰를 붙인다.

- 본인 CLI로 코드 리뷰 실행 → 정해진 JSON 형식(DESIGN 4.2) 수신 → 코멘트로 변환(3→2→1 방식, DESIGN 4.3).
- 기계적 1차 검사(빌드·금지 기능) → 게이트 판정(DESIGN 4.4) → 사용자 수동 머지 → 다음 과제 발급.

**완료 기준:** 코드만 있는 과제를 실제로 리뷰받고, 통과/반려가 게이트로 갈리며, 통과 시 다음 과제가 나온다.

### 마일스톤 3 — 렌더 검증

그래픽스의 핵심 축을 붙인다.

- 코스 가이드라인 파이프라인으로 정답 화면을 생성·검증해 숨겨 보관한다([COURSE_GUIDELINE_PIPELINE.md](COURSE_GUIDELINE_PIPELINE.md)).
- 스타터 템플릿의 캡처 모드 + 서버 캡처 + SSIM 비교 → 렌더 축을 게이트에 반영(DESIGN 4.5).

**완료 기준:** 화면 결과물이 있는 과제에서, 렌더가 정답과 다르면 반려되고 맞으면 통과된다.

### 마일스톤 4 — 커리큘럼·대시보드·온보딩

사용 경험을 완성한다.

- 레퍼런스 기반 커리큘럼 협상 → `syllabus.md` 생성(하드코딩 트랙 대체).
- 1회성 온보딩(LLM CLI·봇 토큰·GitHub 연결).
- 대시보드(스킬 트리 진척, 리뷰 미러, 페르소나 표시) + 레벨 보정 반영.

**완료 기준:** 처음 켜서 설정하고, 직접 정한 주제로 트랙을 만들어 루프를 돌릴 수 있다.

---

## 4. 성공 기준

MVP가 끝났다고 보는 시점은 **사용자가 이 도구만으로 그래픽스 트랙 하나(예: 레이캐스터)를 처음부터 끝까지 완주**하고, 그 과정에서 리뷰·렌더 검증·게이트가 실제로 학습에 도움이 되었을 때다.
